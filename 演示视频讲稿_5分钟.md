# 灵宠家伴 — 5分钟演示视频讲稿

## 第一幕：开场引入（0:00 - 0:25）约80字

**【镜头：正面半身出镜，背景为机器人实物 + 电脑屏幕】**

大家好，我是杨彦浩。今天为大家展示我们的参赛作品——**灵宠家伴**。

这是一个**基于大语言模型Agent的家庭服务机器人系统**。它能听懂你说的话，控制机器人运动，还能在手机、平板、电脑上远程操控。最关键的是——**说话即操控，无需遥控器**。

下面我来演示核心功能。

## 第二幕：AI Agent 对话操控（0:25 - 1:15）约160字

**【镜头：切到电脑屏幕，打开AI对话界面】**
**【操作：进入 AIChat 页面，输入"前进3秒"】**
**【镜头：同时展示对话框 + 机器人实际运动画面（画中画效果）】**

现在展示最核心的功能——**AI Agent 智能操控**。

我直接在对话框输入"前进3秒"——（点击发送，等机器人前进约3秒自动停下）。看，机器人自动前进并停下了。不需要遥控器，不需要按键，就像跟人说话一样。

再来一个更复杂的——"左转然后后退2秒"。
**【操作：输入"左转然后后退2秒"，机器人依次执行】**
AI会自动理解指令链，拆解为多个动作，按顺序执行。

我们还能用AI设置提醒。打字"3分钟后提醒我该休息了"——
**【操作：输入提醒指令，屏幕显示提醒创建成功的提示】**
AI会自动解析提醒内容和时间，到时间后机器人会用语音播报提醒你。

背后的技术是：我们给大语言模型（Kimi K2.5）配置了**function calling工具**——control_robot控制机器人、set_reminder设置提醒。AI理解自然语言后，直接调用这些工具，通过WebSocket把指令传到底层硬件执行。

**【镜头：切到Dashboard主控面板】**

除了AI对话，我们也提供了完整的**手动操控面板**。

所有的机器状态——电量、信号强度、连接状态——都在顶部状态栏实时显示。按下拍照按钮，照片自动保存到相册。

**【操作：点击拍照】**

## 第四幕：鸿蒙端App演示（2:00 - 2:45）约150字

**【镜头：切到华为MatePad Air，打开鸿蒙App】**

接下来是**鸿蒙原生App**的演示。

我们在HarmonyOS上用ArkTS语言开发了完整的移动端应用。大家可以看到，功能和Web端完全一致——方向键控制、云台调节、实时视频画面。

**【操作：在平板上点击方向键，展示机器人响应】**
**【操作：切换到AI对话页面，输入指令】**

AI对话功能也完整移植到鸿蒙端。打字"拍一张照片"，Agent理解后控制机器人拍照并返回照片链接。

**【操作：切换到提醒管理页面、相册页面】**

提醒管理——可以看到刚才AI创建的提醒在这里；相册管理——刚才拍的照片也在这里。删照片、加照片都可以。

鸿蒙原生App共计**6个页面、6个可复用组件**，与Web端共用同一套WebSocket通信协议和REST API，实现了真正的跨端统一体验。

## 第五幕：语音助手 "Hey Jarvis"（2:45 - 3:30）约150字

**【镜头：切到机器人实物，展示麦克风和扬声器，对机器人说话】**

现在展示**本地语音助手**。我们的机器人在Pi上跑了完整的语音流水线。我不需要键盘和鼠标——

**"Hey Jarvis, what time is it?"**
**【等机器人回答】**

它刚才的流程是：唤醒词检测 → 语音识别(Whisper) → 路由到LLM → 语音合成(Piper TTS) → 扬声器播放。全程本地运行（除LLM调用云端Kimi）。

**"Hey Jarvis, tell me the weather."**
**【等机器人回答】**

**"Hey Jarvis, move forward for 2 seconds."**
**【机器人前进】**

注意到了吗？语音指令也能直接控制机器人。这背后的Router会判断意图——是天气查询就走Weather API，是运动指令就走control_robot工具。**一套大脑，多端输入**——Web打字、鸿蒙打字、语音说话，全都要经过同一个AI Agent。

## 第六幕：WebRTC语音通话（3:30 - 4:00）约100字

**【镜头：切回电脑，展示WebRTC通话按钮】**

还有一个重要的功能——**WebRTC实时语音通话**。

**【操作：点击拨号按钮，建立WebRTC连接，对着电脑说话，机器人的扬声器播放声音】**

点击拨号后，浏览器和树莓派之间建立P2P的音频直连，延迟极低。这样你可以通过机器人和家里的人实时对话——相当于一个**可移动的视频电话**。

通话结束后，机器人的语音助手会自动恢复待命。两者共享麦克风资源，通过锁机制协调。

---

## 第七幕：系统架构 + 跨系统协同（4:00 - 4:35）约120字

**【镜头：展示系统架构图（可以是PPT幻灯片或屏幕分享）】**

最后简单介绍一下系统架构，帮大家理解这些功能是怎么串起来的。

我们的系统由**五个模块**协同工作：

**上层**是Web前端(Vue 3)和鸿蒙App(ArkTS)——用户交互入口。**中层**是树莓派上的Python FastAPI实时后端——WebSocket消息中继、MJPEG视频分发、AI对话处理全在这里。**数据层**是Java Spring Boot后端——用户认证(JWT)、提醒调度(@Scheduled)、操作日志。**底层**是STM32嵌入式固件——FreeRTOS实时系统、PID闭环电机控制、编码器速度反馈。

关键的数据流是这样的：你说一句话→Web/鸿蒙/语音→Python后端→Kimi AI理解→返回控制指令→WebSocket→树莓派→UART→STM32→电机PID→轮子转动。整个过程在局域网环境下，端到端延迟不到100毫秒。

## 第八幕：总结（4:35 - 5:00）约90字

**【镜头：回到正面出镜，机器人放在旁边】**

总结一下，**灵宠家伴**做到了三点核心创新：

**第一，具身智能**——AI不只对话，还能控制物理设备，"说话即操控"。**第二，多端协同**——Web、鸿蒙App、语音三种方式统一接入同一AI Agent。**第三，全栈打通**——从Cortex-M4的PID控制到鸿蒙ArkTS界面，从底层硬件到上层应用，全部开源可定制。

我们的目标是让每个家庭都能拥有一个听得懂话、动得了身的智能机器人伙伴。
